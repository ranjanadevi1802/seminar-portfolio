Chapter 01

What is Git & GitHub?

Understanding the difference — and why both matter.

🗂️
Git

A version control system that runs locally on your computer. It tracks every change you make to your files — like a detailed history you can always go back to.

Runs on your machine · No internet needed
☁️
GitHub

A cloud platform where you store your Git repositories online. It lets you collaborate with others, review code, and host your projects publicly or privately.

Lives in the cloud · Built for collaboration
How They Work Together
1
Write Code
You edit files locally on your computer using any editor
2
Commit
Git saves a snapshot of your changes with a message
3
Push
Your commits are uploaded to GitHub for the team to see
4
Collaborate
Others pull your changes, review them, and merge via PRs
Chapter 02

Essential Git Commands

Every command you need — from setup to collaboration.

🔧 Setup — Run once on your machine
$ git config --global user.name "Your Name"
Sets your identity for all commits
$ git config --global user.email "you@email.com"
Links commits to your GitHub account
📁 Start a Repository
$ git init
Initialise a new Git repo in the current folder
$ git clone <url>
Download an existing GitHub repo to your machine
💾 Stage & Commit Changes
$ git status
See which files have changed or are untracked
$ git add .
Stage all changed files ready for commit
$ git add index.html
Stage a specific file only
$ git commit -m "your message here"
Save a snapshot with a descriptive message
$ git log --oneline
View a compact history of all commits
🌿 Branching & Merging
$ git branch
List all local branches
$ git checkout -b feature/ranjana
Create and switch to a new branch
$ git checkout main
Switch back to the main branch
$ git merge feature/ranjana
Merge a branch into the current branch
🔗 Push, Pull & Sync with GitHub
$ git remote add origin <url>
Connect your local repo to a GitHub repo
$ git push origin main
Upload your commits to GitHub
$ git push origin feature/ranjana
Push a specific branch to GitHub
$ git pull origin main
Download and merge the latest changes from GitHub
🔀 What is a Pull Request (PR)?

A Pull Request is a request to merge your branch into main. It lets your teammate review your code, leave comments, and approve before anything is merged. This is how professional teams ship code safely — no one pushes directly to main.

Push your branch
→
Open PR on GitHub
→
Teammate reviews
→
Approve & Merge
Chapter 03

Deploying Your Site

Two free platforms — same repo, two live URLs.

🐙
GitHub Pages
Free · Static sites · yourusername.github.io
1Push index.html to the main branch
2Go to repo → Settings → Pages
3Source: Deploy from a branch → select main
4Click Save — live in ~2 minutes
5Visit: yourusername.github.io/repo-name
✓ Auto-redeploys every time you push to main
🚀
Render
Free · Static sites · yoursite.onrender.com
1Go to render.com → Sign up with GitHub
2Click New → Static Site
3Connect your GitHub repo
4Branch: main · Publish dir: . (root)
5Click Create Static Site — deploys automatically
✓ Supports custom domains & environment variables
GitHub Pages vs Render — Quick Comparison
Feature GitHub Pages Render
Cost Free forever Free tier available
Auto-deploy on push ✅ Yes ✅ Yes
Custom domain ✅ Yes ✅ Yes
Backend / server support ❌ Static only ✅ Yes (Flask, Node…)
Best for Portfolios, docs, HTML sites Full-stack apps too
